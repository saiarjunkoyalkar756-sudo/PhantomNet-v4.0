# Event Schemas

This document outlines the Kafka event schemas used throughout the PhantomNet platform. These schemas define the structure of data exchanged between microservices and agents, enabling real-time processing, analysis, and communication.

---

### 1. `TelemetryEvent`

**Topic:** `telemetry-events`

**Producer:** PhantomNet Agent (via HTTP to `telemetry-ingestor`)

**Consumer:** `event-normalizer`

**Description:** Represents raw telemetry data collected by a PhantomNet agent. This is the initial ingress point for agent-generated data.

**Schema (Pydantic Model):**

```python
from pydantic import BaseModel
from typing import Dict, Any
from uuid import UUID

class TelemetryEvent(BaseModel):
    agent_id: str
    timestamp: str # ISO 8601 format
    event_type: str # e.g., "AGENT_TELEMETRY", "PROCESS_CREATE", "FILE_MODIFY"
    data: Dict[str, Any] # Raw event data
    tenant_id: UUID # Identifier for the tenant this event belongs to
```

**Example Payload:**

```json
{
  "agent_id": "agent-12345",
  "timestamp": "2024-01-01T12:00:00Z",
  "event_type": "PROCESS_CREATE",
  "data": {
    "pid": 1234,
    "name": "cmd.exe",
    "cmdline": "cmd.exe /c whoami",
    "username": "attacker",
    "parent_pid": 567
  },
  "tenant_id": "00000000-0000-0000-0000-000000000001"
}
```

### 2. `NormalizedEvent`

**Topic:** `normalized-events`

**Producer:** `event-normalizer`

**Consumer:** `ai-behavioral-engine` (and potentially others like correlation engine, data lake ingestors)

**Description:** Represents telemetry data after it has been processed by the `event-normalizer` service. It includes standardized fields and a `tenant_id`.

**Schema (Pydantic Model - defined in `phantomnet_agent/orchestrator.py` for now):**

```python
from pydantic import BaseModel, Field
from typing import Dict, Any, Optional
from datetime import datetime
from uuid import UUID # Assuming AIAnalysisResult also uses UUIDs or strings

# AIAnalysisResult schema (simplified for documentation)
class AIAnalysisResult(BaseModel):
    status: str
    risk_score: float
    confidence: float
    threat_category: str
    details: Dict[str, Any]


class NormalizedEvent(BaseModel):
    agent_id: str
    timestamp: datetime # Converted to datetime object
    event_type: str
    payload: Dict[str, Any] # Original event data + normalization fields
    tenant_id: UUID # Propagated tenant_id
    normalized_at: datetime # Timestamp of normalization
    platform_schema_version: str = "1.0.0" # Version of the platform's schema
    ai_analysis_result: Optional[AIAnalysisResult] = None # Enriched by AI engines
```

**Example Payload:**

```json
{
  "agent_id": "agent-12345",
  "timestamp": "2024-01-01T12:00:00Z",
  "event_type": "PROCESS_CREATE",
  "payload": {
    "pid": 1234,
    "name": "cmd.exe",
    "cmdline": "cmd.exe /c whoami",
    "username": "attacker",
    "parent_pid": 567,
    "source_system": "endpoint"
  },
  "tenant_id": "00000000-0000-0000-0000-000000000001",
  "normalized_at": "2024-01-01T12:00:05Z",
  "platform_schema_version": "1.0.0",
  "ai_analysis_result": {
    "status": "analyzed",
    "risk_score": 75.5,
    "confidence": 0.85,
    "threat_category": "Privilege Escalation",
    "details": {"model_version": "v1.2"}
  }
}
```

### 3. `Alert` (Kafka Message)

**Topic:** `alerts`

**Producer:** `ai-behavioral-engine` (and other detection engines)

**Consumer:** `alert-storage`, `websocket_manager` (for real-time frontend display), SOAR engine

**Description:** Represents a detected security alert, generated by various AI or rule-based engines.

**Schema (Inferred from `ai-behavioral-engine/main.py`):**

```python
from uuid import UUID
from datetime import datetime

class AlertMessage: # Not a Pydantic model in source, but represented as dict
    alert_id: str # Unique ID for the alert
    tenant_id: UUID # Identifier for the tenant
    rule_id: str # ID of the rule/model that triggered the alert
    rule_name: str # Human-readable name of the rule
    agent_id: str # Agent that generated the event leading to the alert
    triggered_at: str # ISO 8601 timestamp when the alert condition was met
    severity: str # e.g., "low", "medium", "high", "critical"
    details: str # Human-readable details about the alert
    # Additional fields could be added for richer context
```

**Example Payload:**

```json
{
  "alert_id": "alert-1704067200000",
  "tenant_id": "00000000-0000-0000-0000-000000000001",
  "rule_id": "rule_high_event_frequency_v1",
  "rule_name": "High Frequency Event Anomaly",
  "agent_id": "agent-12345",
  "triggered_at": "2024-01-01T12:00:00.000000Z",
  "severity": "medium",
  "details": "Agent agent-12345 generated 105 events in the last 60 seconds, exceeding the threshold of 100."
}
```

### 4. `AgentCommandPayload`

**Topic:** `agent-commands`

**Producer:** `gateway-service` (via Frontend/API)

**Consumer:** PhantomNet Agent (`orchestrator`)

**Description:** Represents a command issued from the backend (typically by an analyst via the frontend) to a specific agent.

**Schema (Pydantic Model):**

```python
from pydantic import BaseModel
from typing import Dict, Any, Optional
from uuid import UUID

class AgentCommandPayload(BaseModel):
    command_type: str # e.g., "kill_process", "get_file", "run_script", "quarantine_host"
    arguments: Dict[str, Any] = {} # Arguments specific to the command
    task_id: Optional[UUID] = None # Unique ID for tracking command status/response
    tenant_id: UUID # Identifier for the tenant that issued the command
    target_agent_id: str # Specific agent to which the command is directed
    issued_by: str # User or system that issued the command
    issued_at: str # ISO 8601 timestamp when the command was issued
```

**Example Payload:**

```json
{
  "command_type": "kill_process",
  "arguments": {
    "pid": 1234,
    "reason": "Malicious activity detected"
  },
  "task_id": "a1b2c3d4-e5f6-7890-1234-567890abcdef",
  "tenant_id": "00000000-0000-0000-0000-000000000001",
  "target_agent_id": "agent-12345",
  "issued_by": "admin@phantom.net",
  "issued_at": "2024-01-01T12:05:00Z"
}
```
