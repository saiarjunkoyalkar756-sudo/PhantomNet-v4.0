# Database Schemas (PostgreSQL)

This document details the PostgreSQL database schemas used by the PhantomNet platform, focusing on the core tables for multi-tenancy, user management, and alerts.

---

### 1. `tenants` Table

**Description:** Stores information about each tenant (customer) in the multi-tenant SaaS platform. Each user and data record (e.g., alerts) will be associated with a `tenant_id`.

**SQL (from `backend_api/shared/database.py` model converted to SQL):**

```sql
CREATE TABLE tenants (
    id UUID PRIMARY KEY,
    name VARCHAR(255) UNIQUE NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    is_active BOOLEAN DEFAULT TRUE
);
```

**Fields:**

*   `id` (UUID): Unique identifier for the tenant.
*   `name` (VARCHAR): Human-readable name of the tenant (e.g., "Acme Corp").
*   `created_at` (TIMESTAMPTZ): Timestamp when the tenant record was created.
*   `is_active` (BOOLEAN): Flag indicating if the tenant account is active.

---

### 2. `users` Table

**Description:** Stores user authentication and authorization details. Each user is linked to a specific `tenant_id`.

**SQL (from `backend_api/shared/database.py` model converted to SQL):**

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    tenant_id UUID NOT NULL REFERENCES tenants(id),
    username VARCHAR(255) UNIQUE NOT NULL,
    hashed_password VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL,
    twofa_enforced BOOLEAN DEFAULT FALSE,
    totp_secret VARCHAR(255),
    webauthn_enabled BOOLEAN DEFAULT FALSE,
    trust_score REAL DEFAULT 100.0
);
```

**Fields:**

*   `id` (SERIAL): Primary key, auto-incrementing integer.
*   `tenant_id` (UUID): Foreign key referencing the `tenants.id` table, linking the user to their tenant.
*   `username` (VARCHAR): User's unique identifier (e.g., email address).
*   `hashed_password` (VARCHAR): Hashed password for security.
*   `role` (VARCHAR): User's role (e.g., "admin", "analyst", "user").
*   `twofa_enforced` (BOOLEAN): Indicates if 2FA is enforced for this user.
*   `totp_secret` (VARCHAR): Base32 secret for TOTP-based 2FA.
*   `webauthn_enabled` (BOOLEAN): Flag for WebAuthn (FIDO2) support.
*   `trust_score` (REAL): A dynamic score reflecting the user's current trust level based on behavior.

### 3. `alerts` Table

**Description:** Stores security alerts generated by various detection engines (e.g., AI Behavioral Engine). Each alert is associated with a specific `tenant_id`.

**SQL (from `infra/postgres/schema.sql`):**

```sql
CREATE TABLE alerts (
    id SERIAL PRIMARY KEY,
    tenant_id UUID NOT NULL REFERENCES tenants(id),
    alert_id VARCHAR(255) UNIQUE NOT NULL,
    rule_id VARCHAR(255) NOT NULL,
    rule_name VARCHAR(255),
    agent_id VARCHAR(255) NOT NULL,
    triggered_at TIMESTAMPTZ NOT NULL,
    severity VARCHAR(50),
    details TEXT,
    raw_event JSONB,
    received_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Fields:**

*   `id` (SERIAL): Primary key, auto-incrementing integer.
*   `tenant_id` (UUID): Foreign key referencing the `tenants.id` table, linking the alert to its tenant.
*   `alert_id` (VARCHAR): Unique identifier for the alert.
*   `rule_id` (VARCHAR): Identifier for the rule or model that triggered the alert.
*   `rule_name` (VARCHAR): Human-readable name of the triggering rule.
*   `agent_id` (VARCHAR): Identifier of the agent that reported the event leading to the alert.
*   `triggered_at` (TIMESTAMPTZ): Timestamp when the alert condition was met.
*   `severity` (VARCHAR): Severity level of the alert (e.g., "low", "medium", "high", "critical").
*   `details` (TEXT): Detailed description of the alert.
*   `raw_event` (JSONB): Optional, JSON representation of the raw event data that triggered the alert.
*   `received_at` (TIMESTAMPTZ): Timestamp when the alert was received and stored by the backend.

### 4. `session_tokens` Table

**Description:** Stores active and historical session tokens for users, crucial for tracking login sessions, implementing session attestation, and anomaly detection.

**SQL (from `backend_api/shared/database.py` model converted to SQL):**

```sql
CREATE TABLE session_tokens (
    id SERIAL PRIMARY KEY,
    jti VARCHAR(255) UNIQUE NOT NULL,
    user_id INTEGER NOT NULL REFERENCES users(id),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    expires_at TIMESTAMPTZ NOT NULL,
    is_valid BOOLEAN DEFAULT TRUE,
    revoked_at TIMESTAMPTZ,
    ip VARCHAR(255),
    user_agent TEXT,
    device_fingerprint TEXT,
    anomaly_score REAL,
    city VARCHAR(255),
    region VARCHAR(255),
    country VARCHAR(255),
    latitude REAL,
    longitude REAL
);
```

**Fields:**

*   `id` (SERIAL): Primary key.
*   `jti` (VARCHAR): Unique JWT ID (used to invalidate tokens).
*   `user_id` (INTEGER): Foreign key referencing `users.id`.
*   `created_at` (TIMESTAMPTZ): When the session was created.
*   `expires_at` (TIMESTAMPTZ): When the session token expires.
*   `is_valid` (BOOLEAN): Flag indicating if the token is still valid.
*   `revoked_at` (TIMESTAMPTZ): Timestamp if the token was explicitly revoked.
*   `ip` (VARCHAR): IP address from which the session originated.
*   `user_agent` (TEXT): User-Agent string from the client.
*   `device_fingerprint` (TEXT): A unique identifier for the client device.
*   `anomaly_score` (REAL): Score indicating behavioral anomaly at login/session start.
*   `city`, `region`, `country`, `latitude`, `longitude` (VARCHAR/REAL): Geolocation data of the session origin.

### 5. `password_reset_tokens` Table

**Description:** Stores tokens issued for password reset requests.

**SQL (from `backend_api/shared/database.py` model converted to SQL):**

```sql
CREATE TABLE password_reset_tokens (
    id SERIAL PRIMARY KEY,
    token_id VARCHAR(255) UNIQUE NOT NULL,
    user_id INTEGER NOT NULL REFERENCES users(id),
    issued_at TIMESTAMPTZ DEFAULT NOW(),
    expires_at TIMESTAMPTZ NOT NULL,
    used_at TIMESTAMPTZ,
    ip_request VARCHAR(255),
    ip_use VARCHAR(255)
);
```

**Fields:**

*   `id` (SERIAL): Primary key.
*   `token_id` (VARCHAR): The actual password reset token.
*   `user_id` (INTEGER): Foreign key referencing `users.id`.
*   `issued_at` (TIMESTAMPTZ): When the token was issued.
*   `expires_at` (TIMESTAMPTZ): When the token expires.
*   `used_at` (TIMESTAMPTZ): When the token was used.
*   `ip_request` (VARCHAR): IP from which the reset was requested.
*   `ip_use` (VARCHAR): IP from which the reset was confirmed.

---

### 6. `recovery_codes` Table

**Description:** Stores hashed recovery codes for 2FA bypass in case of lost authenticator devices.

**SQL (from `backend_api/shared/database.py` model converted to SQL):**

```sql
CREATE TABLE recovery_codes (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL REFERENCES users(id),
    code_hash VARCHAR(255) NOT NULL,
    used_at TIMESTAMPTZ
);
```

**Fields:**

*   `id` (SERIAL): Primary key.
*   `user_id` (INTEGER): Foreign key referencing `users.id`.
*   `code_hash` (VARCHAR): Hashed recovery code.
*   `used_at` (TIMESTAMPTZ): Timestamp if the code was used.
